# Heroes Battle

Проект представляет собой реализацию пошаговой стратегической игры Heroes Battle.
В рамках проекта были разработаны алгоритмы генерации армии, поиска пути, выбора целей и симуляции боя.

## Реализованные алгоритмы и их сложность

### 1. Генерация армии (`GeneratePresetImpl`)
Алгоритм создает армию компьютера, выбирая наиболее эффективные юниты по соотношению (Атака + Здоровье) / Стоимость.
- **Алгоритм:** Жадный алгоритм с сортировкой.
- **Сложность:** `O(N * log N)`, где `N` — количество доступных типов юнитов.
    - Сортировка списка юнитов занимает `O(N * log N)`.
    - Цикл добавления юнитов выполняется за линейное время относительно лимита очков, что является константой в рамках задачи.

### 2. Симуляция боя (`SimulateBattleImpl`)
Алгоритм управляет ходом сражения, определяя очередность ходов и вызывая атаки юнитов.
- **Алгоритм:** Сортировка и итерация.
- **Сложность:** `O(R * N * log N)`, где `R` — количество раундов, `N` — количество живых юнитов.
    - В каждом раунде происходит сортировка юнитов по инициативе (атаке), что занимает `O(N * log N)`.

### 3. Поиск пути (`UnitTargetPathFinderImpl`)
Алгоритм находит кратчайший путь для юнита к цели, обходя препятствия.
- **Алгоритм:** Поиск в ширину (BFS).
- **Сложность:** `O(W * H)`, где `W` и `H` — ширина и высота игрового поля.
    - В худшем случае алгоритм посещает каждую клетку поля один раз. Так как поле имеет фиксированный размер (27x21), алгоритм работает очень быстро.

### 4. Выбор цели для атаки (`SuitableForAttackUnitsFinderImpl`)
Алгоритм определяет подходящие цели для атаки, выбирая незаблокированных юнитов (передний край).
- **Алгоритм:** Линейный поиск по рядам.
- **Сложность:** `O(R * C)`, где `R` — количество рядов (3), `C` — количество юнитов в ряду.
    - Мы проходим по каждому ряду один раз, пока не найдем живого юнита. Это соответствует проходу по всем клеткам армии.

## Установка и запуск

1. Скачайте файл `heroes_student_task.jar` из этого репозитория.
2. Перейдите в папку с установленной игрой `Heroes Battle`.
3. Откройте папку `jars`.
4. Удалите (или переименуйте) существующий там файл (например, `obf.jar` или старый `heroes_task.jar`).
5. Вставьте скачанный `heroes_student_task.jar`.
6. Запустите игру.

5. ## Требования и стек технологий

* **Язык:** Java 17+
* **Среда разработки:** IntelliJ IDEA
* **Система сборки:** Стандартная сборка IntelliJ IDEA / Maven
* **Тестирование:** JUnit 5.8.1

### Инструкция по установке зависимостей
Проект использует библиотеку **JUnit 5** для модульного тестирования.

**Если вы открываете проект в IntelliJ IDEA:**
1. Проект должен автоматически подгрузить зависимости, если настроен как Maven-проект.
2. Если классы тестов (в папке `test`) горят красным, необходимо подключить библиотеку вручную:
   * Перейдите в `File -> Project Structure -> Modules -> Dependencies`.
   * Нажмите `+` -> `Library...` -> `From Maven...`.
   * Введите в поиск: `org.junit.jupiter:junit-jupiter:5.8.1`
   * Нажмите OK и Apply.
